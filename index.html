About me

Born in Albania, raised in Italy I now live in the States where I know I can best fulfill my ambitions. So what are my ambitions? Well, I consider myself an avid problem solver. This goes beyond my professional life: the way I go about life is by considering it a sequence of different size problems. This has shaped my mind to always look for ways to provide the best solution I can come up with. Therefore, my commitment to computer science and engineering has felt very natural as they make up great tools for problem solving. And the great thing is that it also works the other way: computer science is a great tool for solving problems, but to effectively use computer science you also need to solve problems. This is the perfect playground where computer science not only became my career goal but constitutes my main source of fun in my free time. Game developing, Android mobile programming, AI demonstrations, 3D & 2D rendering software, old console emulators, website development: these are part of the things I engaged in in my free time in these past years. I love technology, developing Software and all you can achieve having these skills. I strongly believe that, theoretical computational limits aside, there's very little things I can't achieve with the skills I possess. I am convinced that having this drive is the premise for a successful career in Computer Science and I am convinced as well that any recruiting company will highly benefit from it. My ambition is to find the right place to build on this drive, I thrive in a motivated and ambitious environment and I look forward, with a lot of excitement, to committing to what is next.

Programming Languages

Paul Graham in his essay about Programming Languages talks about the Blub Paradox to represent that tendency of programmers to settle with a specific programming language, knowing why their favorite programming language (called "Blub" to avoid hurting anyone's feelings) is better than all other programming languages lacking a particular feature they are really comfortable with, but oblivious to what is better thinking that the better programming languages are equivalent to Blub, but they just have a different, more complicated, syntax. Since the programming language shapes the programmer's way of thinking about Software Engineering, they are going to be very resilient to a change. Getting into Computer Science that programming language for me was Java. Java makes programming very easy as strongly oriented at OOP that allows the programmer to focus on the high level design of the code, as well as removing pointers that are a common source of bugs. This comes at a cost: things as the JVM and/or garbage collection may lower performances. This becomes a problem when performances is an important non-functional requirement. Quoting Paul Graham in his essay,

programming languages vary in power. <...> If you have a choice of several languages, it is, all other things being equal, a mistake to program in anything but the most powerful one.
I think this is a very profound principle that distinguish a Software Engineer from a programmer: You can easily write a non trivial client-server application in Java, but most likely writing the same application in C++ may result in a faster program and if we take this one step further, using Rust may improve the safety (in terms of memory and/or concurrency) of the program. In light of this, I've explored many programming languages ideal for the tools they provide for use cases. This made me confident in the process of learning new programming languages & technologies. I have meaningful previous experience with:

  • Rust for system programming and client-server applications
  • Python for data analysis
  • Matlab for analysis in the medical field
  • Java for complex software systems that benefit from high level of abstraction
  • C/C++ for system programming
  • HTML, CSS, Javascript for web development
  • SQL for Relational Databases
  • SmallTalk for didactic GUI application

Having touched on many programming languages, it's hard to keep track of all the different syntactic differences, therefore my current and main programming language is Rust. This ultimately means that on top of the characteristic features of it, Rust is the language I mostly recall the syntax of, as well as its idioms. Regarding the other programming languages, thanks to the strong Software Engineering foundation, I am confident it would be a straightforward and smooth transition to new programming languages.
